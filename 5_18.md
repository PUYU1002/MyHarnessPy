## 一、项目初始目标

搭建一个**实验室自有的、最简练、高可扩展的 LLM Agent Harness**。

核心出发点：

- 实验室需要一个**可以直接改动核心循环**的基础平台，为后续模块创新提供干净的底座
  - 例：设计新型推理策略（Tree-of-Thought、自适应思维链）
  - 例：自主进化（Agent 自我修改工具集或提示词）


设计原则：**最小可用 → 可读性 > 性能 → 每个模块边界清晰、可替换**

## 二、当前实现进度

### 2.1 整体架构

```
用户输入
  │
  ▼
AgentEngine（状态机）
  │  WAITING_INPUT → RUNNING → COMPLETED / WAITING_CONFIRMATION
  ▼
ReactLoop（ReAct 主循环）
  ├─ ContextCompressor（超长上下文自动压缩）
  ├─ LoopDetector（重复工具调用检测）
  ├─ ConfirmationGate（危险操作审批）
  └─ ToolExecutor（并发执行工具）
        ├─ BuiltinTools（12 个内置工具）
        └─ MCPClient（外部工具协议接入）

LLMProvider（可插拔）
  ├─ OpenAI / OpenAI-Compatible
  └─ Anthropic（含流式思考输出）

接入层
  ├─ FastAPI REST + WebSocket
  └─ CLI（终端直连）
```

## 三、Python vs. JavaScript 重构问题

**结论：目前不需要重构，后续也不建议全量重写。**

### 3.1 为什么 OpenCode 这类生产工具选 TypeScript

OpenCode、Cursor、Zed AI 等工具选 TypeScript

| 驱动因素 | 说明 |
|----------|------|
| **运行环境是 IDE** | VS Code 插件、终端 UI（ink）是 JS 原生生态，Python 进不去 |
| **前后端同语言** | CLI 前端 + 后端逻辑合并一个 TS codebase，共享类型定义，减少接口摩擦 |
| **面向开发者工具** | 目标用户是前端/全栈开发者，TS 生态更熟悉 |
| **发行方式** | npm 包分发，开箱即用；Python 分发（pip、venv）对普通用户门槛高 |
| **团队规模大** | TypeScript 强类型在大团队多人协作中防止接口漂移 |

### 3.2 为什么我们用 Python、不需要重构

| 维度 | Python 的优势 |
|------|--------------|
| **LLM SDK** | Anthropic / OpenAI 官方 SDK Python 最完善，typing、async 支持最好 |
| **科研扩展** | 后续接 PyTorch、FAISS、DSPy 等模块无需桥接层 |
| **原型效率** | 相同功能 Python 代码量约是 TS 的 60%，迭代快 |

**合理路径**：Python 保持核心引擎；如需面向外部用户发布或接入 IDE，可在外层加薄 Node.js/Electron 壳，核心逻辑不动。

## 四、代码规模与审阅建议

### 4.1 规模统计

| 部分 | 行数 |
|------|------|
| 核心框架（harness/ + api/） | ~5,000 行 |
| 测试（tests/） | ~2,000 行 |
| 前端（index.html） | ~1,850 行 |
| **合计** | **~8,850 行** |

### 4.2 后续可以继续改进的地方


- **两个 LLM Provider 可以抽一个公共基类。

- **CLI 和 REST 有重复代码**：`cli.py` 和 `api/rest.py` 各自做了一遍 provider 选择和 session 初始化，其实调的是同一个 `build_engine()`，可以合并一下。

- **存储层有功能没用到**：`sqlite.py` 里写了 checkpoint（存档/回滚）功能，但引擎里完全没调用，

- **`todo_write` 工具不应该传 session_id 有点奇怪**：LLM 并不知道自己的 session_id，要靠系统提示告知，容易出错。更好的做法是框架层自动注入，不暴露给 LLM。

## 五、距离"基本 Harness"还差什么

当前已是**可用的 Agent 执行框架**，但作为完整基础平台，尚缺：

| 缺失项 | 说明 
|--------|------|
| **长期记忆** | 当前无向量检索，对话记忆仅靠上下文窗口 
| **沙箱隔离** | shell/powershell 目前直接在宿主机执行，无容器隔离
| **Token 计费统计** | 每次对话消耗 token 数量未记录
