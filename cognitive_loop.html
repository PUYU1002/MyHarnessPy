四环路核心设计原则
感知 — 消息即真相
_messages 是唯一的信息载体，所有外部输入、历史、工具结果、系统指令统一转化为 Message 对象。没有隐式状态传递，一切感知均显式可追踪。
思考 — 工具不缓存
每轮调用 registry.discover() 重新加载工具列表，不依赖缓存。支持运行时动态注册工具，且 LLM 始终看到最新可用能力。
行动 — 安全并发
asyncio.gather() 实现同轮真并发；参数数组防止 shell 注入；硬上限 + OverflowStore 防止 token 爆炸；错误工具不中断整轮循环。
反馈 — 原子性保证
assistant_msg 和 tool_result_msg 必须一次性原子追加，消息协议校验在追加前强制执行。WebSocket 推送与内部回路解耦，各自独立可靠。
