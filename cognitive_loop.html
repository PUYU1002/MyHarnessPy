进化飞轮：反馈结果写回感知层 → 触发下一轮推理 → 能力持续累积
① 感知 · Perception
系统如何观测自身运行状态与能力边界？
新增模块
EventCollector扩展
订阅 EventEmitter 事件流，结构化采集延迟 / Token 用量 / 工具调用频次 / 压缩触发率
FailureClassifier新增
每次会话结束后分类失败模式：LoopDetected · ToolError · ContextOverflow · ToolNotFound · MaxRoundHit
SessionReplayer接 Checkpoint
从 SQLite Checkpoint 逐轮还原历史消息序列，供离线模式挖掘与根因分析
SelfModelStore新增
维护动态自我画像 JSON：擅长任务 / 薄弱环节 / 技能缺口优先级榜单
复用的 harness 设计
observability/EventEmitter storage/sqlite 双表扩展 engine/Checkpoint engine/LoopDetector 指纹
② 思考 · Thinking
系统如何从感知数据推断缺陷并生成改进方案？
新增模块
ArchIntrospector新增
读取 config.yaml + skills/ + personas/，将当前架构序列化为 LLM 可理解的上下文，建立"自我认知"基础
CausalAnalyzer新增
给定失败 trace，让 LLM 推理根因：缺工具 / Prompt 指令不足 / 压缩过早丢失目标 / 循环阈值偏低
CounterfactualReasoner新增
假设"拥有技能 X / 工具 Y"后重新模拟失败会话，验证该假设能否消除失败并量化改善幅度
ProposalSynthesizer新增
输出结构化 Proposal：type(skill|tool|prompt|config) + diff + rationale + risk_level
复用的 harness 设计
AnthropicProvider · Extended Thinking harness/skills.py · load_persona engine/compression.py llm/registry · 统一认证
③ 行动 · Action
系统如何将推理出的方案安全地写入自身能力库？
新增模块
IsolatedRunner新增
在隔离子进程中启动携带 Proposal 的 Agent，重放历史失败会话，观察行为是否改善
SkillWriter写入 skills/
自动生成 / 更新 skills/*/SKILL.md，含 YAML frontmatter + 操作指南；热加载无需重启
PersonaEvolver写入 personas/
追加新技能摘要 / 修正错误指令到 personas/*.md；保留历史版本，仅追加不覆盖
ToolComposer新增
生成新工具 Python 代码草稿，写入 tools/pending/ 待审核目录；人工 review 后手动注册
复用的 harness 设计
skills/ 热加载机制 tools/registry · 动态注册 storage/Checkpoint · 隔离基础 tools/executor · asyncio 安全执行
④ 反馈 · Feedback
系统如何评估改进效果并决定固化或回滚？
新增模块
BehaviorDiff新增
量化新旧版本对比：任务成功率差 / Token 节省 / 平均轮数变化 / 工具调用效率
RegressionGuard接 tests/
自动运行现有测试套件；新版本不得使已通过的用例退化，失败则自动拒绝 Proposal
ApprovalGate新增
risk_level=high → 强制人工确认；通过 REST 端点暴露审核接口；低风险可配置自动通过
EvolutionAuditLog新增
不可删除的进化历史：时间 / Proposal 摘要 / 沙箱指标 / 审批人；供回溯与审计
复用的 harness 设计
storage/Checkpoint · 回滚引擎 storage/session · 审计存储 tests/ · 回归套件 api/rest · 审批端点
↺
④ 反馈验证通过 → SkillWriter / PersonaEvolver 热写入，主循环无感知。 下一轮用户任务触发 EventCollector 继续采集，FailureClassifier 检测新模式， SelfModel 随之更新，进入下一次"感知→思考→行动→反馈"进化飞轮。 整条元认知循环对核心 ReAct 引擎完全透明，可随时降级退出。
